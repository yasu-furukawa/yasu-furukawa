How to join my group

For students seeking admission to the Ph.D program:

I am looking for highly motivated and hard-working graduate students to work with in my group with an ambition to explorer research to change the world. Please apply from our department's application website, then mention my name in the application.

For current Ph.D students or researchers looking for a postdoc position:

I am also looking for a postdoc with strong research background in 3D vision, which usually means multiple first-author papers at premier Computer Vision conferences (CVPR, ICCV and ECCV).

For Wash-U undergraduates:

I am looking for outstanding undergraduate students who are interested in my research. If you would like to know what kinds of research are conducted in my lab, please look at our publications, images, and videos, which should give you a good idea. In general, you should be very good at C++ (Java experience is a good start) with strong math skills to be a good researcher/engineer in 3D Computer Vision. If you are interested, the following is a list of entry projects that should be relatively approachable without Computer Vision knowledge. Currently, there is only one project, but I will add more in the future. Projects have many degrees of freedom and please write code to do something interesting/fun/useful, then drop me an email. We will discuss your results and further opportunities. Of course, if you have already worked on a Computer Vision project from your interests, you can bring that in to start discussion.

Sample entry projects

#1. Analyzing CAD (Computer Aided Design) models


Dealing with 3D models and images is a frequent and essential operation in 3D Vision. Take a look at a CAD model, which people created by a CAD software and shared on the web. Write some code to analyze the model file (simple text format), then visualize analyzed information via 3D model files. Google 3D Warehouse is a repository of CAD models, and this is a link to a CAD model of Colloseum in Rome, which we are often interested in. The model may not be downloadable in a friendly text format, and I already converted the model to a friendly text format .obj with the texture images, which can be found here. This is a link to another Colloseum model in 3D warehouse and the corresponding OBJ file package.

OBJ format is a text format for 3D graphics and very simple. More details about the format can be found here, here or here, for example.

The CAD model in the sketchup format can be opened by the sketchup software (free). Other miscellaneous 3D model formats can be opened by Meshlab for visualization.

The following is an example of what can be done on this 3D data more specifically.
  • Each face of a 3D model is associated with a texture image, whose name is specified in a line somewhere before. Architectural scenes exhibit repetition and you may be able to identify repeating structure by collecting all the faces that are associated with the same texture image. Generate a new OBJ file that only contains such facaes to prove that you could extract a single structure that contains many repetitions. You may be able to also identify the core repeating element, then how many times it is repeated. Repeating structures may not use exactly the same texture image, but often their texture images have very similar filenames, which can be used to identify the entire repeating structure. Generate a file and open it by Meshlab to verify.
  • A CAD model may consist of multiple groups, which a modeler probably generated for organization. You can probably find such group names inside a OBJ file. Write a code to extract 3D geometry, whose group name is X, starts from Y, and etc. Again, generate a new OBJ file and open it by Meshlab to verify.
